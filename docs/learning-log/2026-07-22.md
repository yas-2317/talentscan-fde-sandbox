# 2026-07-22｜JavaScript・Node.js・packageの関係

## 今日の到達点

- JavaScriptとNode.jsの違いを説明できた。
- ブラウザだけで完結する処理と、バックエンドが必要な処理を区別できた。

## JavaScriptとNode.jsの違い

JavaScriptはプログラミング言語、Node.jsはJavaScriptをブラウザ以外で実行するための実行環境である。

ブラウザ内のJavaScriptは、主に画面表示やユーザー操作を扱う。Node.js側では、開発サーバーの起動、API、DB接続、ファイル操作、外部サービス連携などを扱える。

「ブラウザの外で実行する」とは、ChromeやSafariではなく、Macやサーバー上でJavaScriptを実行するという意味である。

## ブラウザだけで完結する処理

サーバーやDBとの通信が不要な処理は、ブラウザ内のJavaScriptだけで実行できる。

- 表示の切り替え
- メニューやモーダルの開閉
- 画面上にすでにあるデータの並び替えや絞り込み
- 入力途中の内容の保持
- 入力値の簡易チェック

判断基準は「データを扱うか」ではなく、「サーバーやDBとの通信が必要か」である。

次の処理にはバックエンド側の処理が必要になる。

- DBからデータを取得する。
- データを保存・更新する。
- Bedrockなどの外部サービスを呼ぶ。
- APIキーやDB接続情報を使う。
- 認証・権限確認を行う。

## packageの役割

packageは、他の開発者が作った機能のまとまりである。このプロジェクトの`dependencies`には、次のpackageが定義されている。

| package | 役割 |
|---|---|
| `next` | Next.js本体。画面、ルーティング、サーバー処理、開発サーバーなどを扱う。 |
| `react` | 画面をコンポーネント単位で作る。 |
| `react-dom` | Reactで作った画面をブラウザに描画する。 |
| `mermaid` | テキストから構成図やフロー図を描画する。 |

## pnpm devで開発サーバーが起動する流れ

`package.json`の`scripts`には、次の設定がある。

```json
"dev": "next dev"
```

そのため、`pnpm dev`を実行すると次の順番で処理される。

```text
pnpm dev
↓
pnpmがpackage.jsonのscripts.devを探す
↓
next devを実行する
↓
Node.js上でNext.jsの開発サーバーが起動する
↓
localhost:3000で画面を確認できる
```

`pnpm dev`はNext.jsを直接起動する特別なコマンドではない。pnpmが`package.json`の`dev`設定を読み、そこに書かれた`next dev`を実行している。

## package関連ファイルの関係

| 名前 | 役割 |
|---|---|
| `package.json` | プロジェクトで使うpackageと実行コマンドを定義する。 |
| `pnpm-lock.yaml` | 実際にインストールするpackageの詳細なバージョンを固定する。 |
| `node_modules` | インストールされたpackageの実体が置かれる。 |
| `pnpm` | packageのインストールと、`package.json`にあるscriptの実行を担当するpackage manager。 |

このプロジェクトのロックファイルは`pnpm-lock.yaml`である。
