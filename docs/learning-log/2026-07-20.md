# 2026-07-20｜OS・ターミナル・プロセス

## 今日の到達点

`pnpm dev`からTalentScanの表示までを、OS、ターミナル、Node.js、Next.js、localhost、ブラウザの役割に分けて説明できるようになった。

## OSとは

macOSはMac全体を管理するOSで、アプリ、メモリ、ファイル、ネットワーク、プロセスが動く土台になる。

## ターミナルとは

ターミナルは、OSやプログラムへ文字で指示を出すアプリである。「OSに指示を出すパネル」から一歩進み、コマンドをシェルへ渡す入口だと理解した。シェルの詳細は今後扱う。

## プロセスとは

プロセスは、OS上で現在動いているプログラムである。ChromeやNext.jsの開発サーバーもプロセスとして管理される。

## Node.jsとは

Node.jsはJavaScriptをブラウザ外で実行する環境である。TalentScanではNode.js上でWebアプリの枠組みであるNext.jsが動く。

## npmとpnpmの違い

どちらも基本的に`package.json`の`scripts.dev`を実行するが、使うパッケージマネージャーが異なる。npmは`package-lock.json`、pnpmは`pnpm-lock.yaml`を使う。このリポジトリは後者があるため、混在させずpnpmへ統一する。

## TalentScanが表示されるまで

```text
ターミナルで pnpm dev
→ Node.js上でNext.jsが起動
→ localhost:3000で待ち受け
→ ブラウザがアクセス
→ Next.jsの返した結果をブラウザが表示
```

ブラウザがNext.jsを起動するのではない。localhostという自分のMac上の宛先へアクセスし、すでに動くNext.jsの結果を表示する。

`http://localhost:3000`を開き、`Ctrl + C`で停止する流れを確認した。

## 理解確認と回答

1. 起動するもの：Node.js上のNext.js。
2. ターミナル：OSやプログラムへ文字で指示を出すアプリ。
3. ブラウザ：起動済みのNext.jsへアクセスし結果を表示する。

3問とも正解した。

## 今日説明できるようになったこと

開発コマンドがMac上にサーバーを起動し、その後ブラウザがlocalhost経由で接続するという役割分担を説明できる。
