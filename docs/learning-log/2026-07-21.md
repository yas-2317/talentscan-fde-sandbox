# 2026-07-21｜ファイル・ディレクトリ・パス

## 今日の到達点

TalentScan内で現在地を確認し、ファイルやディレクトリの役割を説明しながら移動できた。

## ディレクトリとファイル

ディレクトリはファイルや子ディレクトリを整理する箱、ファイルはコード、文章、設定などの実体である。「本棚と本」という理解に、子ディレクトリも置けるという点を加えた。

## パスとは

パスは場所を示す住所である。`/Users/chikamayasufumi/Codex/talentscan-fde-sandbox`は絶対パス、`app/page.tsx`はプロジェクトルートから見た相対パスである。

## 基本コマンド

| コマンド | 意味 |
|---|---|
| `pwd` | 現在地のパスを表示 |
| `ls` | 現在地の中身を表示 |
| `cd` | 指定先へ移動 |

実行するディレクトリによって結果が変わる点も重要である。

## TalentScanの主要ディレクトリ

| 名前 | 役割 |
|---|---|
| `app` | 画面とルーティング |
| `components` | 再利用するUI部品 |
| `lib` | 共通処理やユーティリティ |
| `public` | 画像などの静的ファイル |
| `docs` | 学習ログや資料 |

## package.jsonの役割

`package.json`はライブラリ一覧だけではない。プロジェクト名、バージョン、依存package、実行スクリプトを持つNode.jsプロジェクトの設定ファイルである。`pnpm dev`は`scripts.dev`の`next dev`を実行する。

## README.mdの役割

READMEはルールだけでなく、目的、環境、起動方法、構成、使い方、注意点を人向けにまとめるプロジェクト説明書である。


## 理解確認と回答

1. ディレクトリとファイル：本棚と本。子ディレクトリも入る。
2. `pwd`と`ls`：現在地と、その中身を表示する。
3. `package.json`とREADME：プログラム用の設定と、人向けの説明書。

## 今日説明できるようになったこと
